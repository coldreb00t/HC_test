# HARDCASE FITNESS - Мобильное приложение

## Обзор проекта

HARDCASE FITNESS - это мобильное приложение для фитнеса, которое позволяет тренерам создавать программы тренировок для своих клиентов, а клиентам - отслеживать свои тренировки и прогресс.

## Технологии

Приложение разработано с использованием следующих технологий:

- **React Native** - фреймворк для разработки кроссплатформенных мобильных приложений
- **Expo** - платформа для упрощения разработки React Native приложений
- **TypeScript** - типизированный JavaScript для более надежного кода
- **Supabase** - бэкенд-сервис, предоставляющий API для хранения данных, аутентификации и авторизации

## Функциональность

Приложение предоставляет две роли: клиент и тренер.

### Функциональность для клиентов:
- Просмотр запланированных тренировок
- Отслеживание прогресса выполнения тренировок
- Отметка выполненных упражнений и подходов
- Ведение заметок к упражнениям
- Управление профилем

### Функциональность для тренеров:
- Управление списком клиентов
- Создание и редактирование тренировок для клиентов
- Отслеживание прогресса клиентов
- Управление упражнениями и их параметрами

## Установка и запуск

```bash
# Установка зависимостей
yarn install

# Запуск приложения
yarn start
```

После запуска отсканируйте QR-код с помощью приложения Expo Go на вашем мобильном устройстве.

## Выявленные проблемы и их решения

### 1. Проблемы с конфигурацией окружения

**Проблема:** Несоответствие в использовании переменных окружения в разных файлах.

**Решение:**
- Унифицировали доступ к переменным окружения через `process.env.EXPO_PUBLIC_*` в файле `src/api/supabase.ts`
- Добавили запасные значения для локальной разработки

### 2. Проблемы с потоком аутентификации

**Проблема:** Путаница между автоматическим определением типа пользователя и выбором типа на отдельном экране.

**Решение:**
- Обновили `UserTypeScreen.tsx` для использования унифицированного метода из `AuthContext`
- Модифицировали `determineUserType` в `AuthContext.tsx` для использования сервиса `authService`

### 3. Проблемы конфигурации навигации

**Проблема:** Несоответствия в именах маршрутов и типах навигации.

**Решение:**
- Обновили импорты типов в экранах для использования правильных типов из `src/types/navigation.types.ts`
- Обеспечили согласованность с константами маршрутов из `src/constants/routes.ts`

### 4. Дублирование клиента Supabase

**Проблема:** Две отдельные реализации клиента Supabase в разных файлах.

**Решение:**
- Консолидировали всю логику Supabase в `src/api/supabase.ts`
- Преобразовали `src/lib/supabase.ts` в файл-реэкспортер с предупреждениями об устаревании для обратной совместимости

### 5. Ошибки импорта компонентов

**Проблема:** Некорректное использование компонентов из библиотеки иконок.

**Решение:**
- Исправили импорты и использование `Feather` из `@expo/vector-icons` вместо `Icons.*`

### 6. Несоответствие паттернов доступа к API

**Проблема:** Разные подходы к взаимодействию с Supabase API.

**Решение:**
- Все прямые запросы к API теперь проходят через централизованный файл `src/api/supabase.ts`
- Бизнес-логика вынесена в отдельные сервисы в папке `src/services/`
